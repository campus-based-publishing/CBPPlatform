00016  * new processes can be spawned for a given process type.
00017  *
00018  * The process ID is not an integer but a globally unique string
00019  * identifier that has to fulfill the following additional functions:
00020  * 1) It is used as a one-time-key to authorize the the web
00021  *    request spawning a new process. It therefore has to be
00022  *    random enough to avoid it being guessed by an outsider.
00023  * 2) We also use the process ID as a unique token to implement
00024  *    an atomic locking strategy to avoid race conditions when
00025  *    executing processes in parallel.
00026  *
00027  * We use the uniqid() method to genereate one-time keys. This is not
00028  * really cryptographically secure but it probably makes it difficult
00029  * enough to guess the key to avoid abuse.
00030  * This assumes that we don't start using processes for more sensitive
00031  * tasks. If that happens we'd need to improve the randomness of the
00032  * process id (e.g. via /dev/urandom or similar).
00033  *
00034  * This usage of the processes table also explains why there is no
00035  * updateObject() method in this DAO. If you need a process with different

00040 // Define the max number of seconds a process is allowed to run.
00041 // We assume that no process should run longer than
00042 // 15 minutes. So we clean all processes that have a time
00043 // stamp of more than 15 minutes ago. Running processes should check
00044 // regularly (about once per minute) whether "their" process entry
00045 // is still their. If not they are required to halt immediately.
00046 // NB: Don't set this timeout much shorter as this may
00047 // potentially cause more parallel processes being spawned
00048 // than allowed.
00049 define('PROCESS_MAX_EXECUTION_TIME', 900);
00050

00052 // flooding in case of an error.

00286     * key has not been checked before then this call
00287     * will mark it as used.
00288     * @param $processId string the unique process ID
00289     *  which is being used as one-time-key.

00292    function authorizeProcess($processId) {
00293       $process =& $this->getObjectById($processId);
00294       if (is_a($process, 'Process') && $process->getObliterated() === false) {
00295          // The one time key has not been used yet.
00296          // Mark it as used.
00297          $success = $this->update(
00298             'UPDATE processes
00299              SET obliterated = 1
00300              WHERE process_id = ?',
00301             $processId
00302          );
00303          if (!$success) return false;
00304 
00305          // Only authorize the process if its one-time-key
00306          // has not expired yet.
00307          $minTimestamp = time() - PROCESS_MAX_KEY_VALID;
00308          $authorized = ($process->getTimeStarted() > $minTimestamp);
00309 
00310          // Delete the process entry if the process was
00311          // not authorized due to an expired key.
00312          if (!$authorized) $this->deleteObjectById($processId);
00313 
00314          return $authorized;
00315       }
00316 
00317       // Deny access if the process entry doesn't exist or
00318       // the one-time-key has already been marked used. But don't
00319       // delete the process entry in this case to avoid that
00320       // outsiders can stop processes if they guess a key.
00321       return false;
