00017  * @brief Class that provides meta-data properties compliant with
00018  *  a subset of MODS Version 3.4. We only support those sub-elements
00019  *  we have use-cases (and data) for. We map elements and attributes
00020  *  from the original XML standard to 'element/subelement[@attribute="..."]'
00021  *  property names.
00022  *
00023  *  MODS allows most elements, especially top-level elements to be repeated.
00024  *  We do not implement that full flexibility as we only require repeated
00025  *  elements for translation. This allows us to avoid the considerable extra
00026  *  overhead of handling composite elements in most cases. We essentially
00027  *  translate MODS into a flat key-value list wherever possible. The most
00028  *  notable exception to this rule is the name element which is implemented
00029  *  as a composite. Additional composites can be introduced later if required.
00030  *
00031  *  See <http://www.loc.gov/standards/mods/mods-outline.html>.
00032  *
00033  *  Wherever possible we follow the "Digital Library Federation / Aquifer
00034  *  Implementation Guidelines for Shareable MODS Records", see
00035  *  <https://wiki.dlib.indiana.edu/confluence/download/attachments/24288/DLFMODS_ImplementationGuidelines.pdf>
00036  *
00037  *  NB: This class is an application agnostic base class to be extended
00038  *  by application specific versions of the schema configuring this class
00039  *  via constructor arguments.
00040  */
